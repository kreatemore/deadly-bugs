What's the deadly bug? 

  const nextUrl = window.location.search;

  if (nextUrl.startsWith("http")) {
    throw new Error("No external redirects!")
  } else {
    window.location.replace(nextUrl);
  }
What?
On the surface it looks like we're limited to the current domain, but if we use 
//google.com
the browser will add the current protocol (http or https) to our url, resulting in a redirection to an external domain.
How to prevent?
One form of defense can be to ensure the redirects are local - they start with a single / for example.
