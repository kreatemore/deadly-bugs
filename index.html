What's the deadly bug? 

  is_admin = True

  try:
    is_admin = check_user_is_admin(user)
  except Exception as e:
    logger.log(e.message)

  return is_admin
What?
The initial value of 
is_admin
is an "unsafe default".
If an exception is raised while checking if a user is an admin, the variable will still be 
True
.
How to prevent?
Always err on the safe side, in this example the variable 
is_admin
should have been initialized with 
False
